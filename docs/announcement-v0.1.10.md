# Announcement copy — v0.1.10

## Short post

Released `@diegopetrucci/pi-extensions@0.1.10`.

This update changes `notify` so sound is opt-in by default.

Terminal notifications, desktop notifications, and bell stay on by default, and sound is still available via config.

Install:

```bash
pi install npm:@diegopetrucci/pi-notify
```

or:

```bash
pi install npm:@diegopetrucci/pi-extensions
```

## Discord / forum version

I just shipped `v0.1.10` of my pi extensions package.

This release adjusts the `notify` extension defaults so sound playback is no longer enabled automatically. Terminal notifications, desktop notifications, and the terminal bell still work by default, and sound remains available as an explicit config option. The docs and sample config were also updated to show the new default and a minimal example for turning sound back on.

Install the standalone package with:

```bash
pi install npm:@diegopetrucci/pi-notify
```

Or install the full collection:

```bash
pi install npm:@diegopetrucci/pi-extensions
```
