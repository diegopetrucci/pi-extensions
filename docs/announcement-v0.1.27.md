# Announcement copy — v0.1.27

## Short post

Released `@diegopetrucci/pi-extensions@0.1.27`.

This release improves `triage-comments`: `/triage-comments pr` can auto-detect an existing PR for the current branch, and PR intake can hide resolved and/or outdated inline review comments before selecting comments for read-only triage.

Install:

```bash
pi install npm:@diegopetrucci/pi-extensions
```

or standalone:

```bash
pi install npm:@diegopetrucci/pi-triage-comments
```

Standalone package: `@diegopetrucci/pi-triage-comments@0.1.1`.

## Discord / forum version

`pi-extensions@0.1.27` is out.

This release improves `triage-comments` PR intake. `/triage-comments pr` can detect an existing PR for the current non-`main` branch before asking for a PR number, and fetched PR comments can be filtered to hide resolved inline comments, outdated inline comments, both, or neither before triage selection.

Full collection:

```bash
pi install npm:@diegopetrucci/pi-extensions
```

Standalone package:

```bash
pi install npm:@diegopetrucci/pi-triage-comments
```

The standalone package version is `@diegopetrucci/pi-triage-comments@0.1.1`.
