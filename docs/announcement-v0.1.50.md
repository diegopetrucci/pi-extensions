# Announcement copy — v0.1.50

## Short post

Released `@diegopetrucci/pi-extensions@0.1.50` and `@diegopetrucci/pi-oracle@0.1.14`.

This release refreshes Oracle's model preference lists for current Pi catalogs: Kimi K2.7 Code, MiniMax M3, GLM 5.2, plus new hardcoded provider coverage for `ant-ling`, `nvidia`, and `zai-coding-cn`.

Install:

```bash
pi install npm:@diegopetrucci/pi-extensions
```

or standalone:

```bash
pi install npm:@diegopetrucci/pi-oracle
```

Standalone package: `@diegopetrucci/pi-oracle@0.1.14`.

## Discord / forum version

`pi-oracle@0.1.14` / `pi-extensions@0.1.50` is out.

This release fixes [#2](https://github.com/diegopetrucci/pi-extensions/issues/2) by refreshing Oracle's hardcoded model preferences for current Pi model catalogs. Oracle now prefers Kimi K2.7 Code, MiniMax M3, and GLM 5.2 where available, while retaining older Kimi K2.6, MiniMax M2.7, and GLM 5.1 entries as fallbacks for providers that still expose them.

It also adds provider preference coverage and matrix docs for `ant-ling`, `nvidia`, and `zai-coding-cn`.

Full collection:

```bash
pi install npm:@diegopetrucci/pi-extensions
```

Standalone package:

```bash
pi install npm:@diegopetrucci/pi-oracle
```

The standalone package version is `@diegopetrucci/pi-oracle@0.1.14`.
